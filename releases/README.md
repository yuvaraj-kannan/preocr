# Release Notes

This folder contains versioned release notes for PreOCR. Each release has two files:

- `vX.Y.Z.md` - Human-readable release notes in Markdown format
- `vX.Y.Z.json` - Machine-readable release metadata in JSON format

## Structure

```
releases/
├── v0.4.0.md          # Markdown release notes
├── v0.4.0.json        # JSON release metadata
├── v0.3.2.md
├── v0.3.2.json
└── ...
```

## File Formats

### Markdown Format (`vX.Y.Z.md`)

The Markdown files contain:
- Release date and git tag
- Changes grouped by type (Added, Changed, Fixed, etc.)
- Links to PyPI and GitHub Release artifacts

### JSON Format (`vX.Y.Z.json`)

The JSON files contain structured metadata:
```json
{
  "version": "0.4.0",
  "tag": "v0.4.0",
  "release_date": "2024-12-31",
  "status": "released",
  "changes": {
    "added": [...],
    "changed": [...],
    "fixed": [...]
  },
  "artifacts": {
    "pypi": "https://pypi.org/project/preocr/0.4.0/",
    "github": "https://github.com/.../releases/tag/v0.4.0"
  }
}
```

## Usage

### Generate Release Files

Generate release files for a specific version:
```bash
# Generate for current version (from version.py)
python scripts/generate_release.py

# Generate for specific version
python scripts/generate_release.py v0.4.0
python scripts/generate_release.py 0.4.0
```

Sync all releases from CHANGELOG.md:
```bash
# Generate all missing releases
python scripts/sync_releases.py

# Dry run (see what would be generated)
python scripts/sync_releases.py --dry-run
```

Backfill releases from git tags (when CHANGELOG.md entries are missing):
```bash
# Backfill all missing versions from git tags
python scripts/backfill_releases_from_tags.py

# Backfill specific version
python scripts/backfill_releases_from_tags.py --version v0.7.0
```

### Query Release Information

Get release information for CI/CD or scripts:
```bash
# Get JSON (default)
python scripts/get_release_info.py v0.4.0

# Get Markdown
python scripts/get_release_info.py v0.4.0 --format markdown

# List all available releases
python scripts/get_release_info.py --list
```

## CI/CD Integration

Release files are automatically generated and used in CI/CD workflows:

1. **On Version Bump**: When a version bump PR is merged, release files are automatically generated
2. **GitHub Releases**: The JSON files are used to populate GitHub Release descriptions
3. **PyPI**: Release notes can be used for PyPI package descriptions
4. **Validation**: CI/CD validates that release files exist for new versions

## Manual Editing

Release files are auto-generated from `CHANGELOG.md`, but you can manually edit them if needed:

1. **Markdown files**: Edit directly for custom formatting or additional notes
2. **JSON files**: Edit for custom metadata or artifact links
3. **Regeneration**: Running `generate_release.py` will overwrite manual edits, so be careful

## Workflow

### Standard Release Process

1. Update `CHANGELOG.md` with new version entry
2. Update `preocr/version.py` with new version
3. Create version bump PR
4. On merge, CI/CD automatically:
   - Creates git tag
   - Generates release files
   - Creates GitHub Release
   - Publishes to PyPI

### Backfilling Releases

To generate release files for existing versions:
```bash
python scripts/sync_releases.py
```

This will generate release files for all versions found in `CHANGELOG.md` that don't already have release files.

## File Generation

Release files are generated by parsing `CHANGELOG.md`:

- Version sections are extracted: `## [0.4.0] - 2024-12-31`
- Change sections are parsed: `### Added`, `### Changed`, `### Fixed`, etc.
- List items are collected and structured
- Artifact URLs are generated based on repository configuration

## Maintenance

- **Keep CHANGELOG.md updated**: Release files are generated from CHANGELOG.md
- **Version consistency**: Ensure versions in CHANGELOG.md match git tags
- **Date accuracy**: Release dates come from CHANGELOG.md entries
- **Artifact links**: Automatically generated, but can be manually edited if needed
